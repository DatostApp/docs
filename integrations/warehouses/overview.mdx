## Connecting from a private network

If your database isn't publicly reachable, install the [Tunnel Agent](/integrations/developer/tunnel-agent) in your VPC. Datost connects through the agent without you opening any inbound ports.

## Permissions

<Warning>
  Always use a **read-only** database user for Datost. The product never needs write access.
</Warning>

Datost respects your org's [role-based access](/workspace/roles-and-permissions) on top of database credentials — so individual users can only query tables their role allows, even if the underlying connection sees more.
